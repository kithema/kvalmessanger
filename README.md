# Мессенджер

Веб-приложение для обмена сообщениями в реальном времени. Создано на Go с использованием веб-интерфейса. Предназначено для организации быстрого и безопасного общения между пользователями, поддерживает создание чатов, отправку сообщений.

---

## О проекте

### Почему был создан этот продукт?

Мессенджер появился как практическая реализация современного веб-мессенджера с архитектурой клиент-сервер. Проект объединяет потребность пользователей в мгновенном обмене текстовыми сообщениями с инструментами администратора для управления пользователями и каналами. Разработка велась как комплексный пример полноценного приложения с REST API, веб-сокетами для реального времени.

### Какую проблему он решает?

Приложение предоставляет централизованную и контролируемую среду для общения без зависимости от сторонних облачных сервисов. Оно автоматизирует процессы создания чатов, доставки сообщений, предлагая легкую и настраиваемую альтернативу громоздким корпоративным решениям.

## Быстрая установка и запуск

Вы можете запустить приложение за несколько минут, используя готовые файлы:

### 1. Установите необходимое ПО

1.   Установите Go по ссылке: [golang.org/doc/install](https://golang.org/doc/install)
1.   Установите PostgreSQL по ссылке: [postgresql.org/download](https://www.postgresql.org/download/)

### 2. Настройте базу данных

```bash
# Создайте базу данных
createdb messanger_db

# При первом запуске приложения таблицы создадутся автоматически
```

### 3. Настройте переменные окружения

Создайте файл `.env` в корневой папке:

```env
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=ваш_пароль
DB_NAME=messanger_db
JWT_SECRET=ваш_секретный_ключ
PORT=8080
```

### 4. Запустите приложение

**Запуск бэкенда:**
```bash
cd messanger
go mod tidy
go run main.go
```

**Запуск фронтенда:**
```bash
# Откройте файл index.html в браузере или используйте любой HTTP-сервер, например:
live-server.
```

> **Важно**: Перед запуском убедитесь, что сервис PostgreSQL запущен в вашей системе.

## Возможности

 Мессенджер предоставляет полный цикл для общения:

1.   Система пользователей: Регистрация и авторизация с изоляцией данных
1.   Личные сообщения: Обмен текстовыми сообщениями между двумя пользователями в реальном времени
1.   Групповые чаты: Создание общих каналов для общения нескольких участников
1.   История сообщений: Хранение и просмотр истории переписки

## Использование

### Начало работы

При первом запуске зарегистрируйтесь как новый пользователь, или войдите под существующими данными.

### Главный экран

После входа откроется рабочее пространство со списком чатов, окном сообщений и панелью управления.

### Основные функции (панель навигации)

| Раздел | Описание |
|---|---|
| **Чаты** | Список всех активных диалогов и групповых чатов |
| **Профиль** | Редактирование личных данных и настроек аккаунта |

### Работа с сообщениями

1.  Двойной клик по контакту открывает чат
1.  Все сообщения сохраняются в базе данных и подгружаются при открытии диалога
1.  Новые сообщения приходят мгновенно без необходимости перезагрузки страницы

## Разработка и тестирование

Проект построен на архитектуре MVC с чётким разделением:

1.  Модели: `User`, `Message`, `Chat` и др.
1.  Контроллеры (Handlers): `auth.go`, `chat.go`, `message.go`, `user.go`
1.  Маршруты: Централизованное определение в `main.go`
1.  DAO-слой: Прямая работа с БД через `database/sql`

### Тестирование

Проведено базовое тестирование, охватывающее основные сценарии использования:

| Тест-кейс | Сценарий | Охватываемый функционал |
|---|---|---|
| T1 | Регистрация и вход | Аутентификация, создание профиля |
| T2 | Отправка личного сообщения | Обмен сообщениями, обновление интерфейса |
| T3 | Создание группового чата | Управление группами и участниками |
| T4 | Управление контактами | Добавление и удаление из списка друзей |
| T5 | Просмотр истории | Загрузка и отображение старых сообщений |

Интеграционное тестирование проводилось для проверки корректной работы всех модулей в связке.
